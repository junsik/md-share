# Changelog

## 1.3.1 — 2026-07-17

- 설치자가 지정하는 관리자 비밀번호의 최소 길이를 8자로 정해 다양한 secret 정책을
  지원한다.

## 1.3.0 — 2026-07-17

- 설치자가 지정하는 운영자 ID와 비밀번호로 로그인하는 운영 콘솔을 추가했다.
- 운영자 비밀번호를 브라우저에 저장하지 않고 단일 process memory의 만료되는 HttpOnly
  세션으로 교환한다.
- 운영 콘솔에서 본문 없이 저장 상태와 최근 문서 메타데이터를 확인하고 임의 문서를
  CSRF 보호된 요청으로 삭제할 수 있다.
- 운영자 로그인에 client별·process 전체 요청 제한을 적용한다.
- 자동화용 bearer token은 선택 설정으로 유지하고 관리자 세션과 업로드 권한을 분리했다.

## 1.2.0 — 2026-07-17

- 익명 공유 화면에서 운영자 upload token 입력란을 숨겼다.
- 익명 권한은 문서 생성에만 적용하고 최근 문서 목록과 상태 API는 운영자 token으로
  보호한다.
- 운영자 token으로 임의 문서를 삭제할 수 있으며 만료 변경은 문서별 manageToken만
  허용한다.
- 익명 업로드에 client별·instance 전체 고정 시간창 요청 제한과 `429`, `Retry-After` 계약을
  추가했다.
- 익명 UI, 운영자 권한, 문서 간 관리 권한 격리와 요청 제한 회귀 테스트를 추가했다.

## 1.1.0 — 2026-07-17

- 웹에서 문서를 만들 때 문서별 관리 token을 공유 URL과 분리해 현재 브라우저에 저장한다.
- `My documents`에서 이 브라우저가 소유한 문서의 공개 메타데이터 새로 고침, 만료 변경,
  영구 삭제와 로컬 권한 제거를 지원한다.
- 공유 문서 접속자에게 관리 UI나 관리 권한을 노출하지 않는다.
- 브라우저 저장 실패 시에만 일회성 관리 token 복구 UI를 제공한다.
- 실행 가능한 Playwright 사용자 가이드와 결정적 스크린샷 검증을 CI에 추가한다.

## 1.0.0 — 2026-07-16

- 본문과 메타데이터의 원자적 파일 저장 및 고아 파일 정리를 추가했다.
- `Idempotency-Key` 기반 재시도 안전 업로드와 안정적인 오류 code를 추가했다.
- 원본 `.md` 파일명, UTF-8, NUL 문자와 2 MiB 제한 검증을 추가했다.
- 웹 편집기에 `.md` 파일 선택과 drag-and-drop을 추가했다.
- 한 번만 반환되는 문서 관리 token으로 만료 변경과 삭제를 지원한다.
- 공개 메타데이터, 안전한 저장 상태 API와 OpenAPI 3.1 계약을 추가했다.
- 저장소, API와 입력 검증 자동화 테스트를 추가했다.
