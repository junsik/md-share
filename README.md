# md-share

Self-hostable markdown rendering and sharing service. Upload a markdown document, get back a
shareable link that renders it as a clean HTML page — useful when your chat/messenger can't
render `.md` attachments (e.g. reports generated by AI agents).

- Split-pane editor with live preview, `.md` file open/drop, local draft autosave, and expiry choice
- GitHub-flavored markdown (tables, task lists), syntax-highlighted code blocks, mermaid diagrams
- Raw HTML in markdown is **not** rendered (safe default against XSS) — with one
  exception: `<br>` becomes a real line break, since it is the only way to break
  a line inside a GFM table cell
- Atomic filesystem storage, retry-safe uploads, per-document lifecycle control, single small Docker image

## API

| Method & path | Auth | Description |
| --- | --- | --- |
| `POST /api/documents` | Bearer token | Upload markdown, returns the share URL |
| `GET/PATCH/DELETE /api/documents/{id}` | public / management token | Metadata, expiry update, deletion |
| `GET /d/{id}` | none | Rendered HTML page |
| `GET /api/documents/{id}/raw` | none | Raw markdown download |
| `GET /api/documents` | Bearer token | List recent documents |
| `GET /api/status` | Bearer token | Safe storage counters |
| `GET /` | none | Paste-and-share form for manual use |
| `GET /skill.md` | none | Agent skill with this instance's URL substituted in |
| `GET /api.md` | none | This API reference with the instance's URL substituted in |
| `GET /openapi.yaml` | none | OpenAPI 3.1 contract |

Full reference — request/response schemas, error codes, retention semantics,
what renders: **[docs/API.md](docs/API.md)**.

Installation, upgrade, backup, restore, and security operations: **[docs/OPERATIONS.md](docs/OPERATIONS.md)**.

Product milestones and their open-source evidence: **[docs/ROADMAP.md](docs/ROADMAP.md)**.

### Upload examples

```bash
# JSON body
curl -X POST "$BASE_URL/api/documents" \
  -H "Authorization: Bearer $MD_SHARE_UPLOAD_TOKEN" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: $REQUEST_KEY" \
  -d '{"markdown": "# Report\n\nHello", "filename": "report.md", "ttlDays": 30}'

# Raw file body (title/ttlDays as query params)
curl -X POST "$BASE_URL/api/documents?filename=report.md&ttlDays=30" \
  -H "Authorization: Bearer $MD_SHARE_UPLOAD_TOKEN" \
  -H "Content-Type: text/markdown" \
  -H "Idempotency-Key: $REQUEST_KEY" \
  --data-binary @report.md
```

The initial `201` response contains document metadata, `url`, `rawUrl`,
`replayed: false`, and a one-time `manageToken`. A safe retry returns the same
document with `200` and `replayed: true`. Send `url` to your messenger.

## AI agents

md-share is built to be driven by AI agents (report bots, coding assistants), and the
repo ships the integration pieces:

- **The AI button** (top right in the web UI) — per-agent, copy-paste install commands
  for the skill below, with your instance's URL already filled in.
- **[skills/md-share/SKILL.md](skills/md-share/SKILL.md)** — an installable agent skill
  (Claude Code, Codex, OpenCode, or any agent that reads instruction files). It teaches
  the agent when to upload instead of attaching `.md` files, the API call, TTL etiquette,
  and the authoring rules that render well. A running instance serves it at
  `GET /skill.md` with the instance URL substituted, so installing is one command:

  ```bash
  mkdir -p .claude/skills/md-share && \
    curl -fsSL https://your-instance.example.com/skill.md -o .claude/skills/md-share/SKILL.md
  ```

  (Codex/OpenCode: use `.agents/skills/` instead; any other agent: paste the file into
  its instructions.)
- **[docs/API.md](docs/API.md)** — the full API reference, written to be dropped into an
  agent's context as-is
- **[openapi.yaml](public/openapi.yaml)** — the machine-readable v1.0 API contract
- **[llms.txt](llms.txt)** — index for tools that discover docs via the
  [llms.txt](https://llmstxt.org) convention

## Configuration

| Env var | Default | Description |
| --- | --- | --- |
| `MD_SHARE_UPLOAD_TOKEN` | (unset) | Bearer token for upload/list APIs. **Required in production** — uploads are rejected (503) when unset. In development, auth is skipped when unset. |
| `MD_SHARE_ALLOW_ANONYMOUS_UPLOADS` | `false` | Set to `true` to skip upload auth entirely. Only for trusted/internal networks. |
| `MD_SHARE_DATA_DIR` | `./data` (`/data` in Docker) | Document storage directory. Mount a volume for persistence. |
| `MD_SHARE_PUBLIC_BASE_URL` | (derived from request) | Base URL used in returned links; set it when running behind a reverse proxy that doesn't forward the public host. |
| `MD_SHARE_DEFAULT_TTL_DAYS` | (keep forever) | Default retention when a request has no `ttlDays`. |

### Retention

Each upload can set its own `ttlDays` (the web UI offers 1/7/30 days or forever, defaulting to
30 days). When a request has no `ttlDays`, `MD_SHARE_DEFAULT_TTL_DAYS` applies; when that is
also unset, documents are kept forever. Expired documents return 404 immediately and an hourly
background sweep deletes their files, so unvisited links don't accumulate on disk.

## Run

```bash
# Development
pnpm install
pnpm dev            # http://localhost:3000
pnpm test
pnpm lint
pnpm typecheck

# Docker (published image)
docker run -p 3000:3000 -v md-share-data:/data \
  -e MD_SHARE_UPLOAD_TOKEN=change-me ghcr.io/junsik/md-share:latest

# Docker (local build)
docker build -t md-share .
docker run -p 3000:3000 -v md-share-data:/data \
  -e MD_SHARE_UPLOAD_TOKEN=change-me md-share
```
