# md-share

Self-hostable markdown rendering and sharing service. Upload a markdown document, get back a
shareable link that renders it as a clean HTML page — useful when your chat/messenger can't
render `.md` attachments (e.g. reports generated by AI agents).

- Split-pane editor with live preview, local draft autosave, and a share dialog with expiry choice
- GitHub-flavored markdown (tables, task lists), syntax-highlighted code blocks, mermaid diagrams
- Raw HTML in markdown is **not** rendered (safe default against XSS)
- Filesystem storage, optional TTL-based expiry, single small Docker image

## API

| Method & path | Auth | Description |
| --- | --- | --- |
| `POST /api/documents` | Bearer token | Upload markdown, returns the share URL |
| `GET /d/{id}` | none | Rendered HTML page |
| `GET /api/documents/{id}/raw` | none | Raw markdown download |
| `GET /api/documents` | Bearer token | List recent documents |
| `GET /` | none | Paste-and-share form for manual use |

### Upload examples

```bash
# JSON body
curl -X POST "$BASE_URL/api/documents" \
  -H "Authorization: Bearer $MD_SHARE_UPLOAD_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"markdown": "# Report\n\nHello", "title": "Hourly report", "ttlDays": 30}'

# Raw file body (title/ttlDays as query params)
curl -X POST "$BASE_URL/api/documents?title=Report&ttlDays=30" \
  -H "Authorization: Bearer $MD_SHARE_UPLOAD_TOKEN" \
  -H "Content-Type: text/markdown" \
  --data-binary @report.md
```

Response: `{ "id", "title", "createdAt", "expiresAt", "size", "url", "rawUrl" }` — send `url`
to your messenger.

## Configuration

| Env var | Default | Description |
| --- | --- | --- |
| `MD_SHARE_UPLOAD_TOKEN` | (unset) | Bearer token for upload/list APIs. **Required in production** — uploads are rejected (503) when unset. In development, auth is skipped when unset. |
| `MD_SHARE_ALLOW_ANONYMOUS_UPLOADS` | `false` | Set to `true` to skip upload auth entirely. Only for trusted/internal networks. |
| `MD_SHARE_DATA_DIR` | `./data` (`/data` in Docker) | Document storage directory. Mount a volume for persistence. |
| `MD_SHARE_PUBLIC_BASE_URL` | (derived from request) | Base URL used in returned links; set it when running behind a reverse proxy that doesn't forward the public host. |
| `MD_SHARE_DEFAULT_TTL_DAYS` | (keep forever) | Default retention when a request has no `ttlDays`. |

### Retention

Each upload can set its own `ttlDays` (the web UI offers 1/7/30 days or forever, defaulting to
30 days). When a request has no `ttlDays`, `MD_SHARE_DEFAULT_TTL_DAYS` applies; when that is
also unset, documents are kept forever. Expired documents return 404 immediately and an hourly
background sweep deletes their files, so unvisited links don't accumulate on disk.

## Run

```bash
# Development
pnpm install
pnpm dev            # http://localhost:3000

# Docker (published image)
docker run -p 3000:3000 -v md-share-data:/data \
  -e MD_SHARE_UPLOAD_TOKEN=change-me ghcr.io/junsik/md-share:latest

# Docker (local build)
docker build -t md-share .
docker run -p 3000:3000 -v md-share-data:/data \
  -e MD_SHARE_UPLOAD_TOKEN=change-me md-share
```
